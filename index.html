MYSQL Dienst starten/beenden

Als erstes muss man den MySQL Dienst starten. Damit benutzt man den Befehtl über das Init-Skript. Nägmlich: '/etc/init.d/mysql'. Nachdem man den Befehl eingegeben hat, kommt eine Bestätigung, dass der MySQL-Server gestartet werden konnte. Von nun an läuft der Dienst selber. Das bedeutet, dass wenn man das Terminal schliesst, läuft der MySQL Dienst im Hintergrund weiter. Das ist zum Beispiel vergleichbar mit einer Antivierensoftware. Wenn man das Fenster schliesst, läuft die Anwendung trotzdem weiter.

Nun muss man noch die Konsole starten, damit man mit MySQL programmieren kann. für dies gibt man ein 'mysql -u root -p'. Hierbei steht '-u' für 'user' und '-p' dafür, damit das Passwort abgefragt werden soll. Danach folgt logischerweise eine Passworteingabe.

Man kann das Passwort für den MySQL Dienst auch beliebig ändern. Für dies, ist folgender Befehl notwendig: 'set password for root@localhost=Password('DeinPasswort');

Erstellen einer Datenbank

Eine neue Datenbank erstellt man ganz einfach mit: 'create database archiv;'. Nun hat man eine neue Datenbank erstellt und sie, wie man sieht, nach 'archiv' benannt. Nun kann man diese Datenbank (die jetzt natürlich leer ist) anzeigen lassen mit: 'show databases;'. Nun wird eine Liste angezeigt mit allen Datenbanken, die auf diesem Computer bei MySQL registriert sind. In diesem Fall nur eine, nähmlich 'archiv'. Alle anderen, die zu sehen sind, werden automatisch beim Download von MySQL erstellt und sind für diesen Dienst notwendig.


Erstellen einer Tabelle

In einer Datenbank befindet sich immer einer oder mehrere Tabelle/n. Ohne diese können ja alle Daten nicht eingetragen werden. Nun erstellen wir eine Tabelle in der Datenbank "archiv", die wir gerade erstellt haben.

Bevor wir dies tun können, müssen wir allerdings zuerst die Datenbank "archiv" auswählen. Dies funktioniert mit "use archiv;". Danach "create table akten (aktenID smallint primary key auto_increment, persoenlich VARCHAR(50));"

"aktenID" und "persoenlich" sind die so genannten Attribute der Tabelle bzw. Spalten-Namen. "smallint" und "VARCHAR(50)" sind die Tatentypen der Attribute. "smallint" dient zu allen Zahlen, die kleiner als 32767 sind. Das bedeutet, dass in dieser Spalte (akteID) alle Zahlen hineingeschriebern werden können die kleiner als 32767 sind. "primary key" bedeutet, dass über (aktenID) die Tabelle später mit einem "foreign key" verlinkt werden kann. Zusätzlich wird dem Attribut noch "auto_increment" zugeteilt. Dies bezweckt, dass in der ganzen Spalte (aktenID) zufällige Zahlen generiert werden.

"VARCHAR(50)" sagt dem Attribut, dass in der ganzen Spalte maximal 50 Zeichen, Buchstaben oder Zahlen hineingeschrieben werden können.

Datentypen und deren Funktion

VARCHAR
Steht für Texte, die keine fixe Länge besitzen wie z.B Kommentare / VARCHAR(50) = maximal 50 Zeichen

CHAR
Steht für Texte, die eine fixe Länge besitzen wie z.B Ja oder NEIN. / CHAR(4) = maximal 4 Buchstaben (nein)

SMALLINT
Für alle Zahlen, die sich unter 32767 befinden / smallint = <32767

YEAR
Für Jahreszahlen wie "1990" oder "2019" / YEAR

INT / INTEGER
Steht für alle Zahlen, beliebige Grösse / INT / INTEGER

User erstellen und Rechte geben

Einen MySQL User erstellen:

CREATE USER hans@localhost IDENTIFIED BY 'passwort';

"@localhost" steht für lokal bzw. hans = lokaler Benutzer, "IDENTIFIED BY 'passwort'" = Passwortvergabe des Benutzers.

Nun kann man diesem Benutzer noch beliebige Rechte hinzufügen. Zum Beispiel alle Rechte:

GRANT ALL ON archiv.* to hans@localhost IDENTIFIED BY 'rootpasswort' WITH GRANT OPTION;


Datensätze importieren

In eine Tabelle gehören logischerweise auch Daten. Da die ungerne in SQL selber hineingeschrieben werden, importiert man sie meistens aus einer Excel-Tabelle.

Im folgendem Befehl werden Daten vom Ordner "Downloads" in die Tabelle "akten" importiert.

LOAD DATA LOCAL INFILE '/home/m105/Downloads/akten.sql' INTO table akten FIELDS TERMINATED BY ';' ;

Nun kann man mit "select * from akten ;" alle Daten, die gerade importiert wurden anzeigen lassen.

Hierbei steht "FIELDS TERMINATED BY ;" dafür, dass die einzelnen Zeilen und Spalten in der Datei durch ";" getrennt sind. Nur so kann MySQL die Datensätze richtig lesen und importieren.

(INNER) JOIN

Der INNER JOIN oder auch nur "JOIN" ist der, der am meisten verwendet wird. Er ist dafür da, um mehrere Attribute zu selektieren. Das funktioniert in erster Linie ganz eifnach. Wie man unten im Syntax sieht wird als erstes das Attribut angewählt, welches dann später relevant sein wird. Danach wird geschrieben, von welcher Tabelle man das Attribut zieht und zu welcher es verlinkt werden soll (JOIN B also Tabelle B). Wie man oben im Schlüsselpaar sieht, gibt es zwei verschiedene Keys. Der Primary-(Gelb) und der Foreign key(Rot). Es muss immer der primary key mit dem Foreign key verlinkt werden und nicht umgekehrt. Ansonsten funktioniert das ganze nicht. Sprachschichten in SQL

Jeder Befehl in MySQL gehört zur irgend einer Sprachschicht. Eine Sprachschicht beinhaltet eine handvoll Kommandos zum erstellen von Datenbanken, erstellen von Tabellen, Bearbeiten einer DB/Tabelle, Vergabe von Benutzerrechten, Hinzufügen oder Bearbeiten von Datensätzen und um Datensätze auszulesen. Hier geht es zu den 4 verschiedenen Sprachschichten:

  • Data Definition Language
  • Data Query Language
  • Data Control Language
  • Data Manipulation Language

Data Definition Language

Die Data Definition Language umfängt drei Hauptbefehle: ALTER, CREATE und DROP. ALTER wird benötigt, um Datenbanken und Tabellen zu bearbeiten. Mit CREATE erstelle man DB und TB und mit DROP kann man diese löschen. Folgende Syntax zeigen die Funktion:

Die Data Query Language ist für und nur die Select-Anfragen zuständig. Sie ist zugleich die Wichtigste Sprachschicht in MySQL. Die Abfragen können zum Teil sehr anspruchsvoll sein, vorallem, wenn zwei "Select" Befehle in einem Befehl vorkommen. Dies wird HIER beim Full Outer Join gezeigt. "Group By" steht für eine Gruppierung. z.B kann man Group By verwenden bei Postleitzahlen oder Strassen. "Having" benutzt man, wenn eine Bedingung besteht. z.B: Zeig mir alle Personen an, die von der Strasse "x,y" kommen. "Order By" ist für die Ordnung der Ausgabe zuständig. Also ob das Resultat aufgerundet oder abgerundet werden soll, oder nach welchem Attribut es sortiert werden soll. "DESC." bedeutet, dass die Abfrage bzw. Ausgabe absteigend sortiert werden soll.

Funktionen und Operatoren

Gruppenfunktionen

COUNT() Anzahl Datensätze deren Feld nicht leer ist
AVG() Durchschnitt des Wertes in der Klammer
MIN() Kleinster Wert der Klammer
MAX() Grösster Wert der Klammer
SUM() Summe der Klammer

Mathematische Funktionen

RAND() Zufällige Zahl zwischen 0 und 1
ROUND(x,n) Rundet auf x auf n Stellen
MOD() Gibt den ganzzahligen Rest aus

String Funktionen

CONCAT() Zeigt alles in der Klammer in einer Spalte an
LCASE(), UCASE() Verwandelt einen String in Kleinbuchstaben/Grossbuchstaben
LEFT(x,n) n Zeichen von x, von links

Datums Funktionen

MONTH() Monat des Datums in der Klammer
YEAR() Jahr des Datums in der Klammer
DAYOFWEEK() Wochentag des Datums in der Klammer (6=Sonntag, 0=Montag)
